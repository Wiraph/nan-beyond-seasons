# คำอธิบายผลงาน — Nan Game On

## ชื่อผลงาน
**Nan Game On — น่านเล่นได้ทั้งปี** (แพลตฟอร์ม Social Sports Tourism)

## กลุ่มเป้าหมาย
1. **นักกีฬาสมัครเล่น/สายแอคทีฟ** (วิ่ง ปั่น เทรล ล่องแก่ง) — กลุ่มที่เดินทางตามอีเวนต์ ไม่สนฤดูกาล และใช้จ่ายสูงต่อทริป
2. **คนเชียร์และครอบครัว** — ผู้ชมงานประเพณีแข่งเรือยาว งานวิ่ง ที่มากับนักกีฬาและเที่ยวต่อ
3. **ผู้ประกอบการชุมชน** (โฮมสเตย์ ร้านอาหาร คาเฟ่ ผู้ให้บริการล่องแก่ง) — ได้ช่องทางเข้าถึงนักท่องเที่ยวผ่านฟีดและระบบส่วนลดแลกแต้ม

## ปัญหา
การท่องเที่ยวน่านกระจุกตัวในฤดูหนาว ทั้งที่กีฬาและกิจกรรมเด่นที่สุดหลายอย่างของน่าน
**มีเฉพาะนอกฤดูท่องเที่ยวหลัก**: ล่องแก่งลำน้ำว้าระดับ 3–5 ต้องรอน้ำหลากหน้าฝน (ก.ค.–ต.ค.)
ประเพณีแข่งเรือยาวหัวพญานาคจัดช่วงปลายฝน (ก.ย.–ต.ค.) — ไม่มีแพลตฟอร์มที่รวมปฏิทิน
บอกจังหวะเวลา และเปลี่ยนผู้มาร่วมงานให้กลายเป็นนักท่องเที่ยวค้างคืนที่ใช้จ่ายกับชุมชน

## ขั้นตอนการนำ AI มาแก้ปัญหา
1. **AI Race-cation Planner** — ผู้ใช้กดปุ่มเดียวที่หน้างานแข่ง ระบบส่ง (ก) ข้อมูลงาน (ข) พยากรณ์อากาศน่านจริง 7 วันจาก Open-Meteo (ค) สถานที่ท่องเที่ยว/ร้านชุมชนใกล้สนามที่คำนวณระยะทางจริง เข้า LLM แล้วบังคับตอบเป็น STRICT JSON — ได้แผน 2 วัน "แข่ง/เชียร์ + เที่ยว + กิน + พักชุมชน" ที่**ปรับตามอากาศจริง** (ฝนบ่าย → จัด outdoor ไว้เช้า ย้ายเข้าวัด/พิพิธภัณฑ์ช่วงบ่าย) ทุก id ถูก validate กับฐานข้อมูลก่อนแสดง ป้องกัน hallucination
2. **AI Concierge 2 บอท** — "คู่หูสายกีฬา" ตอบเรื่องงานแข่ง+ที่เที่ยวโดย ground ด้วยปฏิทินกีฬาและสถานที่จริง พร้อมการ์ดกดไปหน้างาน/เช็คอินได้ทันที และ "ผู้ช่วยแอป" ตอบวิธีใช้งานโดย ground จากกติกาแต้ม/แบดจ์ในโค้ดจริง
3. **ความทนทาน** — ใช้ free-model chain ของ OpenRouter ไล่ลำดับอัตโนมัติเมื่อติด rate limit และมี rule-based fallback ทุกจุด ระบบใช้งานได้เสมอแม้ AI ไม่พร้อม

## กลไกกระจายรายได้สู่ชุมชน (Gamification)
เช็คอินที่งานกีฬาผ่าน QR → สะสมแต้ม + แบดจ์ตามฤดู (เช่น "นักสู้สายฝน" ต้องมาหน้าฝน,
"Nan All-Season Athlete" ต้องมาครบ 3 ฤดู) → แต้มแลกส่วนลดร้านค้า/โฮมสเตย์ชุมชนที่ร่วมรายการ
→ ฟีด social แบบ Strava (โพสต์/kudos ข้ามเครื่องจริงผ่าน Supabase) สร้างชุมชนและ FOMO
ให้คนกลับมาเก็บแบดจ์ฤดูถัดไป = **เที่ยวซ้ำนอกฤดูโดยธรรมชาติ**

หน้า **แลกแต้ม (`/rewards`)** ปิด loop ธุรกิจให้ครบ: ผู้ใช้เอาแต้มแลกสิทธิ์จริงจากร้าน/โฮมสเตย์/
ผู้ให้บริการล่องแก่งชุมชน แล้วได้โค้ดไปแสดงที่ร้าน — เป็นช่องทางที่ผู้ประกอบการวัดผลได้ว่านักท่องเที่ยว
มาจากแคมเปญ (เวอร์ชัน prototype ยังไม่ตัดแต้ม/ผูกร้านจริง)

**กันเช็คอินปลอม:** เวอร์ชันจริงจะผูก QR แต่ละจุดกับพิกัด GPS ณ สนามแข่ง ต้องอยู่ในบริเวณงานจริง
จึงเช็คอินได้ (prototype ใช้การจำลองสแกน แจ้งชัดในหน้าเช็คอิน)
